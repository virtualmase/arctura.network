Protocol

How the subnet works

Arctura Base extends Bittensor's incentive mechanism to verifiable Base chain intelligence. Miners compete to produce the most accurate, fastest attestations. Validators score on correctness. The metagraph distributes TAO emissions to honest work, trustlessly, every epoch.

01
⬡
Miners Register
Any participant registers a hotkey on the subnet by burning TAO. Miners connect their own Base RPC endpoint — Alchemy, QuickNode, or self-hosted op-geth.
Permissionless
02
⬢
Validators Query
Validators issue AttestationRequest synapses with a target block range. Miners have a defined response window to return proofs.
BFT Scoring
03
◈
Proofs Verified
Validators cross-check Merkle proofs against public Base chain data. Hash matches, latency, and proof integrity are weighted in the Resonance scoring function.
Trustless
04
◆
TAO Emitted
Each epoch, Yuma consensus aggregates validator weights to compute final miner rankings. TAO flows to performance. Alpha tokens capture subnet network value.
τ TAO + α Alpha
Miner Role
What miners do
A miner is a long-running Python process connected to a live Base RPC endpoint. When a validator sends an AttestationRequest, the miner reads the specified block range, constructs a Merkle proof of the data, and returns a signed AttestationResponse synapse. Miners are ranked by accuracy first, latency second.
→ Register hotkey on subnet (burn TAO)
→ Connect a reliable Base RPC endpoint
→ Serve an axon for incoming validator queries
→ Return verified Merkle proofs within the response window
→ Earn TAO proportional to rank each epoch
Validator Role
How Resonance BFT scores
Validators query the full miner set with identical block ranges each epoch. The Resonance scoring function weights each response on three dimensions: hash correctness (0.6), proof integrity (0.3), and response latency (0.1). Validators commit weights to the Bittensor metagraph via Yuma consensus.
Hash match weight 0.60
Proof integrity weight 0.30
Response latency weight 0.10
Missing response penalty score = 0
Consensus mechanism Yuma / Resonance BFT

Known Limitations
⚠ Testnet only. No mainnet netuid yet. All testing should use --network test --netuid 0 until the mainnet slot is confirmed.
⚠ RPC rate limits. Public Base RPC endpoints have strict rate limits. Builders running at scale should use a dedicated Alchemy or QuickNode endpoint.
⚠ AgentKit not yet merged. The AgentKit branch is in active development. Core attestation works independently of AgentKit — miners can register and score without it.
⚠ Scoring parameters may change. The Resonance weight coefficients (hash: 0.6, proof: 0.3, latency: 0.1) are under review before mainnet freeze.

Installation Start Here
1
Clone the repository
Pull the subnet codebase from GitHub.
bash 
$ git clone https://github.com/bittensaur/arctura-base-subnet
$ cd arctura-base-subnet
2
Install dependencies
Python 3.11+ required. A virtual environment is recommended.
bash 
$ python -m venv .venv # optional but recommended
$ source .venv/bin/activate
$ pip install -r requirements.txt
3
Run the test suite
All 42 tests should pass before you proceed. A failing test suite means your environment is not configured correctly.
bash 
$ pytest tests/ -v
# Expected: 42 passed, 0 failed
4
Create a Bittensor wallet
You'll need a coldkey and hotkey to register on the subnet.
bash 
$ btcli wallet new_coldkey --wallet.name arctura_base
$ btcli wallet new_hotkey --wallet.name arctura_base --wallet.hotkey miner
Run a Miner Testnet
1
Set your Base RPC endpoint
Create a .env file in the project root with your RPC URL. Alchemy or QuickNode recommended for reliability.
.env 
BASE_RPC_URL=https://base-mainnet.g.alchemy.com/v2/YOUR_KEY
BASE_CHAIN_ID=8453
2
Start the miner on testnet
bash 
$ python -m neurons.miner \
  --netuid 0 \
  --network test \
  --wallet.name arctura_base \
  --wallet.hotkey miner \
  --logging.debug
3
Verify attestations are being sent
Watch the debug log for AttestationResponse sent entries. Each entry confirms a Merkle proof was returned to a validator query.
bash 
# Watch for proof activity in logs
$ tail -f ~/.bittensor/logs/miner.log | grep "AttestationResponse\|merkle\|proof"
Run a Validator Stake Required
1
Requirements
Validators need a funded coldkey with minimum stake. A stable server with 16+ GB RAM is recommended. Validators must remain online to set weights each epoch or their influence decays.

$ python -m neurons.validator \
  --netuid 0 \
  --network test \
  --wallet.name arctura_base \

Verify scoring and weight-setting

# Watch for scoring events
$ tail -f ~/.bittensor/logs/validator.log | grep "scoring\|weights\|resonance"

# Confirm weights are being committed on-chain
$ btcli su check_set_weights --netuid 0 --network test

Managed providers (recommended)

2

Run your own Base node for maximum reliability and no rate limits. Requires ~2TB SSD for a full node.
bash 
# Run an OP Stack Base node via Docker
$ docker run -p 8545:8545 \
  ethereumoptimism/op-geth \
  --network base-mainnet \
  --http \
  --http.addr 0.0.0.0
3
Verify RPC is responding
bash 
$ curl -X POST $BASE_RPC_URL \
  -H "Content-Type: application/json" \
  --data '{"jsonrpc":"2.0","method":"eth_blockNumber","params":[],"id":1}'
# Returns: {"result":"0x14d3e00",...}
